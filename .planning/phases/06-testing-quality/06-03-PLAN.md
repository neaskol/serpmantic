    2. Fix the failing test or the test infrastructure (e.g., missing mock, import path issue)
    3. Re-run until all tests pass

    Common issues to watch for:
    - Mock collision: two test files mocking the same module differently. Solution: ensure each test file's vi.mock is self-contained in a describe block or use beforeEach/afterEach to reset.
    - NextResponse mock from setup.ts may conflict with actual Next.js imports in route tests. Solution: if route handler tests need real NextResponse, remove the global mock by calling `vi.unmock('next/server')` at the top of those test files and import NextResponse directly.

    - Path alias resolution: vitest.config.ts already has `@` alias. If new test files can't resolve imports, verify the alias is correct.

    After all tests pass, examine the coverage output. Record:
    - Total lines covered percentage
    - Total functions covered percentage
    - Total branches covered percentage
    - Total statements covered percentage

    The current vitest.config.ts has thresholds at 80% which will fail since we're targeting 30-40%. Update thresholds:

    Set coverage thresholds to 5% below actual measured values (floor, not ceiling):
    - If measured lines = 35%, set threshold to 30%
    - If measured functions = 32%, set threshold to 27%
    - If measured branches = 28%, set threshold to 23%
    - If measured statements = 34%, set threshold to 29%

    This creates a ratchet: coverage can only go up from here.

    Run the suite one final time with `npx vitest run --coverage` to confirm thresholds pass.
