<task type="auto">
  <name>Task 2: Create Context Builder for prompt enrichment</name>
  <files>
    apps/web/src/lib/ai/context-builder.ts
  </files>
  <action>
1. Create `apps/web/src/lib/ai/context-builder.ts`:

2. Define and export the `PromptContext` interface:
   ```typescript
   export interface PromptContext {
     keyword: string
     language: string
     semanticTerms: { term: string; minOccurrences: number; maxOccurrences: number }[]
     termsToAvoid: string[]
     structuralBenchmarks: Record<string, { min: number; max: number }>
     currentContent?: string
     selectedText?: string
     userContext?: {
       audience?: string
       tone?: string
       sector?: string
       brief?: string
     }
   }
   ```
   Import `SemanticTerm` and `SerpAnalysis` from `@/types/database` for type reference.

3. Export `buildPromptContext` function:
   - Signature: `(serpAnalysis: SerpAnalysis | null, semanticTerms: SemanticTerm[], guide: { keyword: string; language: string }, options?: { selectedText?: string; currentContent?: string; userContext?: PromptContext['userContext'] }) => PromptContext`
   - Transforms database types into the PromptContext shape
   - Extracts terms-to-avoid by filtering `semanticTerms.filter(t => t.is_to_avoid)`
   - Maps semantic terms to `{ term, minOccurrences, maxOccurrences }` (only non-avoided terms)
   - Uses `serpAnalysis.structural_benchmarks` or empty object if null

4. Export `buildPrompt` function:
   - Signature: `(template: string, context: PromptContext) => string`
   - Replaces template variables using pattern `{variable_name}`:
     - `{keyword}` -> context.keyword
     - `{language}` -> context.language
     - `{semantic_terms}` -> JSON.stringify of top 20 semantic terms (sorted by importance desc)
     - `{terms_to_avoid}` -> comma-separated list
     - `{structural_benchmarks}` -> JSON.stringify
     - `{content}` -> context.currentContent or ''
     - `{selected_text}` -> context.selectedText or ''
     - `{audience}` -> context.userContext?.audience or ''
     - `{tone}` -> context.userContext?.tone or ''
     - `{sector}` -> context.userContext?.sector or ''
     - `{brief}` -> context.userContext?.brief or ''
   - Use a single regex replacement: `template.replace(/\{(\w+)\}/g, (_, key) => replacements[key] ?? '')`

5. Export `buildSystemMessage` function:
   - Signature: `(taskType: string, context: PromptContext) => string`
   - Produces a system prompt string in RCCF format (Role-Context-Constraint-Format)
   - Include SERP-aware instructions:
     - "Target keyword: {keyword}"
     - "Language: {language}"
     - "Important semantic terms to incorporate: {top 10 terms}"
     - "Terms to AVOID (not correlated with ranking): {termsToAvoid}"
   - If userContext.audience exists, add "Target audience: {audience}"
   - If userContext.tone exists, add constraint "Write in a {tone} tone"
   - Add universal constraint: "Do not exceed the semantic benchmarks. Over-optimization (score > 100) should be avoided."
   - This function is a convenience wrapper â€” the executor will use it for prompts without custom system prompts

IMPORTANT: Separate system prompt from user content (security â€” prevents prompt injection). User text goes in the `prompt` parameter, system instructions go in the `system` parameter of streamText.
  </action>
