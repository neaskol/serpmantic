2. Import dependencies:
   - `streamText` from `ai`
   - `getModel` from `./registry`
   - `getModelForTask`, `TaskType`, `DEFAULT_MODEL` from `./router`
   - `buildPrompt`, `buildSystemMessage`, `PromptContext` from `./context-builder`
   - `logger` from `@/lib/logger`

3. Define and export `ExecuteOptions` interface:
   ```typescript
   export interface ExecuteOptions {
     modelId: string           // Full model ID like 'anthropic/claude-sonnet-4-20250514'
     prompt: string            // The enriched user prompt
     systemPrompt?: string     // Optional system prompt (overrides default)
     maxTokens?: number        // Default 4096
     onFinish?: (event: { text: string; usage: { promptTokens: number; completionTokens: number; totalTokens: number }; finishReason: string }) => void | Promise<void>
   }
   ```
   The `onFinish` callback allows the route handler to inject post-stream logic (e.g., writing to the database) while keeping the executor generic.

4. Define and export `ExecuteResult` type:
   ```typescript
   export type ExecuteResult = Awaited<ReturnType<typeof streamText>>
   ```

5. Export `executePrompt` async function:
   - Signature: `(options: ExecuteOptions) => Promise<ExecuteResult>`
   - Implementation:
     a. Resolve the model using `getModel(options.modelId)` from registry
     b. Call `streamText()` from `ai` package with:
        - `model`: resolved model
        - `system`: options.systemPrompt (if provided)
        - `prompt`: options.prompt
        - `maxTokens`: options.maxTokens || 4096
        - `onFinish`: callback that logs completion info AND invokes the caller's onFinish:
          ```typescript
          async onFinish({ text, usage, finishReason }) {
            logger.info('AI execution completed', {
              modelId: options.modelId,
              promptTokens: usage.promptTokens,
              completionTokens: usage.completionTokens,
              totalTokens: usage.totalTokens,
              finishReason,
              textLength: text.length,
            })
            // Invoke caller's onFinish (e.g., route handler writes to DB)
            if (options.onFinish) {
              await options.onFinish({ text, usage, finishReason })
            }
          }
          ```
        - `onError`: callback that logs errors:
          ```typescript
          onError({ error }) {
            logger.error('AI stream error', {
              modelId: options.modelId,
              error: error instanceof Error ? error.message : String(error),
            })
          }
          ```
        IMPORTANT: The executor's `onFinish` wraps both logging AND the caller-provided callback. This allows the route handler to pass its DB write logic as the `onFinish` callback, which runs in the same closure where the Supabase client and user context are available.
     c. Return the streamText result object directly (caller uses `.toTextStreamResponse()`)

6. Export `estimateCost` utility function:
   - Signature: `(modelId: string, usage: { promptTokens: number; completionTokens: number }) => number`
   - Hard-code current pricing in a const object:
     ```typescript
     const PRICING: Record<string, { input: number; output: number }> = {
       'anthropic/claude-sonnet-4-5-20250929': { input: 3.0, output: 15.0 },
       'anthropic/claude-sonnet-4-20250514': { input: 3.0, output: 15.0 },
       'openai/gpt-4o': { input: 2.5, output: 10.0 },
       'openai/gpt-4o-mini': { input: 0.15, output: 0.60 },
     }
     ```
   - Calculate: `(usage.promptTokens * pricing.input + usage.completionTokens * pricing.output) / 1_000_000`
   - Return 0 if model not found in pricing

IMPORTANT:
- Do NOT add custom retry logic. The AI SDK's `streamText` does not have built-in retry for the streaming call itself. If retries are needed, they should be handled at the route handler level before calling executePrompt. Keep the executor focused on a single execution.
- Do NOT use `generateText` — we always stream for consistent behavior. Non-streaming callers can collect the full stream.
- Use `prompt` parameter (string), NOT `messages` array, since our use case is single-prompt execution (not multi-turn chat).

   CRITICAL: Without this, Next.js serverless functions timeout after 10s, truncating AI responses.

4. Import dependencies:
   - `NextRequest`, `NextResponse` from `next/server`
   - `z` from `zod`
   - `createClient` from `@/lib/supabase/server`
   - `executePrompt`, `estimateCost` from `@/lib/ai/executor`

   - `handleApiError`, `generateRequestId` from `@/lib/error-handler`
   - `logger` from `@/lib/logger`

5. Define the request validation schema using Zod:
   ```typescript
   const ExecuteRequestSchema = z.object({
     promptId: z.string().uuid(),
     guideId: z.string().uuid(),
     selectedText: z.string().optional(),
     scope: z.enum(['selection', 'document']).default('document'),
   })
   ```

6. Implement the `POST` handler:
   a. Generate requestId via `generateRequestId()`, set on logger
   b. Create Supabase client via `await createClient()`
   c. Check authentication:
      ```typescript
      const { data: { user } } = await supabase.auth.getUser()
      if (!user) return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
      ```
   d. Parse and validate request body with `ExecuteRequestSchema.parse(body)`
   e. Load prompt from database:
      ```typescript
      const { data: prompt, error: promptError } = await supabase
        .from('prompts')
        .select('*')
        .eq('id', promptId)
        .single()
      ```
      Return 404 if not found. Use typing that matches the prompts table schema.
   f. Load guide + SERP data:
      ```typescript
      const { data: guide, error: guideError } = await supabase
        .from('guides')
        .select('keyword, language, content, prompt_context')
        .eq('id', guideId)
        .single()
      ```
      Return 404 if not found.
   g. Load SERP analysis + semantic terms for this guide:
      ```typescript
      const { data: serpAnalysis } = await supabase
        .from('serp_analyses')
        .select('*')
        .eq('guide_id', guideId)
        .order('created_at', { ascending: false })
        .limit(1)
        .single()

      const { data: semanticTerms } = await supabase
        .from('semantic_terms')
        .select('*')
        .eq('serp_analysis_id', serpAnalysis?.id || '')
      ```
   h. Build prompt context using `buildPromptContext(serpAnalysis, semanticTerms || [], guide, { selectedText, currentContent: guide.content ? JSON.stringify(guide.content) : '', userContext: guide.prompt_context })`
   i. Determine model: Use `prompt.model_id` if set, otherwise `getModelForTask(prompt.task_type as TaskType)`
   j. Build the enriched prompt: `buildPrompt(prompt.prompt_template, promptContext)`
   k. Build system prompt: Use `prompt.system_prompt` if the prompt has one, otherwise `buildSystemMessage(prompt.task_type, promptContext)`
   l. Call `executePrompt` with the `onFinish` callback that writes to the database:
      ```typescript
      const result = await executePrompt({
        modelId,
        prompt: enrichedPrompt,
        systemPrompt,
        maxTokens: 4096,
        onFinish: async ({ usage, finishReason }) => {
          // This runs AFTER the stream completes, inside streamText's onFinish
          // The Supabase client and user are captured in the route handler's closure
          await supabase.from('ai_requests').insert({
            user_id: user.id,
            guide_id: guideId,
            prompt_id: promptId,
            model_id: modelId,
            prompt_tokens: usage.promptTokens,
            completion_tokens: usage.completionTokens,
            total_tokens: usage.totalTokens,
            estimated_cost: estimateCost(modelId, usage),
            finish_reason: finishReason,
          })
        },
      })
      ```
      ARCHITECTURE NOTE: The route handler passes its DB write as the `onFinish` callback to `executePrompt`. The executor wraps this callback inside `streamText`'s `onFinish` (see Task 1), so it runs after the stream completes in the executor's context, but has access to the route handler's closure variables (supabase, user.id, guideId, etc.).
   m. Return `result.toTextStreamResponse()`
      IMPORTANT: Use `toTextStreamResponse()` (raw text stream), NOT `toUIMessageStreamResponse()` (chat protocol). Our client reads raw text via fetch + ReadableStream, not the useChat hook.
   o. Wrap entire handler in try/catch, use `handleApiError(error, { route: '/api/ai/execute', context: { requestId } })` for errors.

7. Follow the existing API route patterns from `apps/web/src/app/api/serp/analyze/route.ts`:
   - Same error handling structure
   - Same logger usage
   - Same Supabase client creation pattern
