1. Create `apps/web/src/lib/ai/outline-builder.ts` with:

   a) `OutlineSection` type re-exported from database.ts (or import from there)

   b) `buildOutlinePrompt(keyword, competitors, topTerms)` function:
      - Parameters:
        - `keyword: string` — target keyword
        - `competitors: Array<{ url: string; title: string; headings: ExtractedHeading[] }>` — SERP pages with optional headings
        - `topTerms: string[]` — top 30 semantic terms by importance
      - Returns: `string` (the full prompt for Claude)
      - Prompt structure uses XML tags (Claude 4.x best practice):
        - `<task>` — Generate optimal H2/H3 outline for keyword
        - `<competitor_headings>` — Only include competitors that have headings (length > 0). If NO competitors have headings, include `<competitor_titles>` section with just URL+title pairs so Claude has context about what's ranking
        - `<semantic_terms>` — Top 30 terms to distribute (2-4 per section)
        - `<guidelines>` — 8 rules: 4-8 H2 sections, 0-4 H3 per H2, distinct aspects, user intent questions, semantic term distribution, specific headings, strict hierarchy, prioritize multi-competitor topics
        - `<anti_patterns>` — Bad vs good heading examples using the actual keyword
        - `<output_format>` — JSON array with level, title, keywords fields. "Return ONLY valid JSON. No markdown, no explanations."
      - The prompt must work in French AND English (the keyword language determines output language). Add instruction: "Generate headings in the same language as the keyword."

   c) `validateOutlineHierarchy(outline: OutlineSection[]): boolean` function:
      - Returns false if outline is empty
      - Returns false if first item is not h2
      - Returns false if any h3 appears before any h2
      - Returns true otherwise

   d) `parseOutlineResponse(responseText: string): OutlineSection[]` function:
      - Strip markdown code fences: `response.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim()`
      - `JSON.parse()` the cleaned text
      - Validate it's an array
      - Validate each item has level ('h2' | 'h3'), title (string), keywords (string[])
      - Throw descriptive error if any validation fails

2. Create `apps/web/src/app/api/ai/plan/route.ts`:

   - `export const maxDuration = 30` — prevent timeout
   - Zod schema: `{ guideId: z.string().uuid() }`
   - Use `generateText()` from 'ai' (NOT streamText — outline generation is fast, <5 sec)
   - IMPORTANT: Import `generateText` from 'ai' package directly, NOT from executor.ts

   Route flow:
   a) Auth check (same pattern as /api/ai/execute)
   b) Parse + validate request body with Zod
   c) Load guide from DB: `supabase.from('guides').select('keyword, language').eq('id', guideId).single()`
   d) Load SERP analysis: `supabase.from('serp_analyses').select('id').eq('guide_id', guideId).order('created_at', { ascending: false }).limit(1).single()`
   e) If no SERP analysis: return 400 `{ error: 'No SERP analysis found. Run analysis first.' }`
   f) Load SERP pages: `supabase.from('serp_pages').select('url, title, score, headings, is_excluded').eq('serp_analysis_id', serpAnalysis.id)` — filter out excluded pages in JS, filter for pages with headings.length > 2 (or use all pages if none have headings)
   g) Load semantic terms: `supabase.from('semantic_terms').select('display_term, importance, is_to_avoid').eq('serp_analysis_id', serpAnalysis.id)` — filter !is_to_avoid, sort by importance desc, slice top 30, map to display_term strings
   h) Build prompt with `buildOutlinePrompt()`
   i) Call `generateText()` with:
      - `model: getModel('anthropic/claude-sonnet-4-5-20250929') as any` (same type cast pattern from executor.ts)
      - `system: 'You are an expert SEO content strategist specializing in competitive SERP analysis. Generate structured H2/H3 outlines optimized for semantic coverage and search intent.'`
      - `prompt: buildOutlinePrompt(...)` result
   j) Parse response with `parseOutlineResponse(result.text)`
   k) Validate hierarchy with `validateOutlineHierarchy(outline)` — if invalid, return 500 with descriptive error
   l) Log to ai_requests table (same pattern as /api/ai/execute onFinish):
      - model_id, prompt_tokens, completion_tokens, total_tokens, estimated_cost
      - prompt_id: null (outline generation is not prompt-based)
   m) Return `NextResponse.json({ outline })` on success
   n) Wrap everything in try/catch with `handleApiError()`

   Error handling:
   - JSON parse failure: return 500 `{ error: 'Generated outline was malformed. Please try again.' }`
   - Validation failure: return 500 `{ error: 'Generated outline has invalid structure. Please try again.' }`
   - Auth failure: 401
   - Guide not found: 404
   - No SERP: 400
