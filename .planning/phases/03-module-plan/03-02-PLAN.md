Rewrite `apps/web/src/components/analysis/plan-panel.tsx` to replace the current placeholder implementation. The component must:

**State management:**
- `generating: boolean` — loading state during API call
- `outline: OutlineSection[] | null` — generated outline data
- `showPreview: boolean` — controls Dialog visibility
- `error: string | null` — error message from generation

**Imports:**
- Import `OutlineSection` from `@/types/database`
- Import `useGuideStore` (guide, score, serpPages, semanticTerms)
- Import `useEditorStore` (editor, plainText)
- Import base-nova Dialog components (check actual exports from dialog.tsx)
- Import Button, Badge, Card, CardContent, Separator from their respective ui/ files
- Import toast from 'sonner'
- Import Lucide icons: FileText, Sparkles, AlertTriangle, BookOpen, ChevronDown, ChevronUp, AlertCircle

**Generate button behavior:**
- Disabled when `generating` is true
- Disabled when `!guide` (no guide loaded)
- Disabled when `editorHasContent` (plainText.trim().length > 50)
- Disabled when `serpPages.length === 0` (no SERP analysis)
- Shows spinner animation during generation
- On click: calls `handleGenerate()`

**handleGenerate() function:**
- Set `generating = true`, `error = null`
- Call `POST /api/ai/plan` with `{ guideId: guide.id }`
- On success: set `outline = data.outline`, `showPreview = true`
- On error: set `error = errorMessage`, show `toast.error(errorMessage)`
- Finally: set `generating = false`

**Preview Dialog (modal):**
- Open when `showPreview === true`
- Title: "Apercu du plan genere" (or similar French label)
- Body: Scrollable list of outline sections
  - Each section shows Badge (H2 = default variant, H3 = secondary variant) + title text
  - H3 sections indented with `pl-4`
  - H2 sections have `font-semibold`
  - Keywords shown as outline Badge components (max 3 per section)
- Footer: Two buttons
  - "Rejeter" (outline variant) — calls `handleReject()`
  - "Accepter & Inserer" (default variant) — calls `handleAccept()`
- On close (X or escape): calls `handleReject()`

**handleAccept() function:**
1. Check `editor` is not null (toast error if null)
2. Check `outline` is not null
3. **Over-optimization check (rough client-side estimate):**
   NOTE: This is an approximate heuristic, NOT a precise scoring calculation. It provides a directional warning to prevent obvious over-optimization. Accuracy is sufficient for a "proceed with caution" signal.

   3.1: Get `semanticTerms` from `useGuideStore`. Filter for terms where `!is_to_avoid` (only include scoring-positive terms). If `semanticTerms` is empty or undefined, skip the over-optimization check entirely (no data to estimate from).

   3.2: Normalize all outline section titles to lowercase and remove accents using a normalize helper:
   ```typescript
   const normalizeText = (text: string) =>
     text.toLowerCase().normalize('NFD').replace(/[\u0300-\u036f]/g, '')
   ```
   Concatenate all normalized outline titles into a single string (join with space).

   3.3: For each filtered semantic term, normalize the term's `display_term` (or `term` field) using the same normalizeText helper. Check if the normalized term appears in the concatenated normalized outline string. Count total matches (each term counts at most once).

   3.4: Multiply match count by 3 (rough estimate: each term match in headings adds ~3 occurrences when the section is written out). Add to current `score` from `useGuideStore`.

   3.5: If `estimatedScore > 100`, show `window.confirm()` with French warning:
   "Attention : ce plan pourrait augmenter votre score semantique a environ ~{estimatedScore} (estimation approximative). Le seuil de sur-optimisation est 100. Voulez-vous continuer ?"
   If user cancels: return (don't insert).

4. Convert outline to HTML: map each section to `<h2>title</h2>` or `<h3>title</h3>`, join with newline
5. Insert into editor: `editor.chain().focus().insertContent(html).run()`
6. Close dialog: `setShowPreview(false)`, `setOutline(null)`
7. Show success toast: `toast.success('Plan insere dans l\'editeur')`

**handleReject() function:**
- `setShowPreview(false)`
- `setOutline(null)`
- `toast.info('Plan rejete')`
