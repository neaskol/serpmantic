**Step 1: Create `apps/web/src/lib/ai/json-extractor.ts`**

Create a utility function `extractJSON(text: string): unknown` that:
1. First tries `JSON.parse(text)` directly
2. If that fails, extracts JSON from markdown code blocks: regex `/```(?:json)?\s*(\{[\s\S]*?\})\s*```/`
3. If that fails, finds the first `{...}` block: regex `/\{[\s\S]*\}/`
4. If nothing works, throws `Error('No valid JSON found in AI response')`

Export as named export `extractJSON`.

**Step 2: Create `apps/web/src/app/api/ai/intention/route.ts`**

Create a POST route handler following the exact pattern of `/api/ai/execute/route.ts`. Key details:

- `export const maxDuration = 30` (critical for Vercel)
- Validate request with Zod schema:
  ```typescript
  const RequestSchema = z.object({
    keyword: z.string().min(1),
    language: z.string(),
    serpPages: z.array(z.object({
      url: z.string(),
      title: z.string(),
    })).min(3, 'At least 3 SERP pages required for intent analysis'),
  })
  ```
- Authenticate with Supabase `createClient()` + `auth.getUser()`
- Return 401 if no user
- Build SERP context string from serpPages: numbered list with title + URL
- Use `executePrompt()` from `@/lib/ai/executor` with:

Request JSON output format matching the `IntentResult` type from intention-panel.tsx
  - `systemPrompt`: "You are an expert SEO analyst specializing in search intent classification. Analyze SERP data to determine user intent. Always return valid JSON without markdown code blocks."
- IMPORTANT: Do NOT use streaming for this route. Use `await result.text` to get the full text, then parse with `extractJSON()`. Return `NextResponse.json(parsed)`.
- Use `onFinish` callback to log to `ai_requests` table (same pattern as /api/ai/execute but with `prompt_id: null` since this isn't a prompt-based execution)
- Error handling: Use `handleApiError()` from error-handler.ts

The prompt template should be in French context since the app targets French SEO:
```
Analyze the search intent for the keyword: "${keyword}"

Based on these top Google results:
${serpContext}

Classify the search intent. For each of these 4 intent types, provide a percentage (must sum to 100%), a description, and 2-3 example user questions:

1. Informationnel - User wants to learn, understand, or find information
2. Transactionnel - User wants to buy, subscribe, download, or take action
3. Navigationnel - User wants to find a specific website or page
4. Comparatif - User is researching/comparing before purchase

Return ONLY valid JSON (no markdown, no explanation):
{

  "confidence": 85,
  "intents": [
    {
      "type": "informationnel",
      "percentage": 70,
      "description": "Users want to understand...",
      "questions": ["What is X?", "How does X work?"]
    }
  ]
}
```

- Validate with Zod:
  ```typescript
  const RequestSchema = z.object({
    keyword: z.string().min(1),
    language: z.string(),
    content: z.string().min(50, 'Content must be at least 50 characters'),
    intents: z.array(z.enum(['informationnel', 'transactionnel', 'navigationnel', 'comparatif'])).min(1),

  ```

- Use `executePrompt()` with `getModelForTask('intent_analysis')` (same Claude model)
- Prompt asks Claude to analyze the content (truncated to first 2000 chars) against the identified intents:

  - Which intents are missing
  - 3-5 specific improvement suggestions
- Request JSON output matching `ContentAnalysis` type from intention-panel.tsx:
  ```json
  {
    "coversIntents": true,
    "matchedIntents": ["informationnel"],
    "missingIntents": ["comparatif"],
    "suggestions": ["Add comparison section...", "Include pricing details..."]
  }
  ```
- Use `extractJSON()` to parse response
- Use `onFinish` callback to log to ai_requests (prompt_id: null)
- Error handling: `handleApiError()`
