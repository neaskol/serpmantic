**Gap 2: Optimistic save UX (from audit: "No optimistic updates")**

Make the save button show immediate feedback:
1. After calling `setSaving(true)`, optimistically show success state in the UI
2. On error: revert metaTitle/metaDescription to previous values and show error toast
3. Implementation approach:
   - Before the fetch, capture `const prevTitle = guide.meta_title ?? ''` and `const prevDesc = guide.meta_description ?? ''`
   - On fetch failure (`!res.ok` or catch block), call `setMetaTitle(prevTitle)` and `setMetaDescription(prevDesc)` to revert
   - This is a lightweight optimistic pattern: the local fields already show the new value (user typed them), and we only revert on failure

The key addition to `handleSave` is:

```typescript
async function handleSave() {
  if (!guide) return
  setSaving(true)

  // Capture previous values for rollback
  const prevTitle = guide.meta_title ?? ''
  const prevDesc = guide.meta_description ?? ''

  try {
    const res = await fetch(`/api/guides/${guide.id}`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        metaTitle: metaTitle,
        metaDescription: metaDescription,
      }),
    })

    if (res.ok) {
      const updatedGuide = await res.json()
      setGuide(updatedGuide)   // <-- NEW: sync guide-store
      toast.success('Meta tags sauvegardees')
    } else {
      // Revert on failure
      setMetaTitle(prevTitle)
      setMetaDescription(prevDesc)
      toast.error('Erreur lors de la sauvegarde')
    }
  } catch {
    // Revert on network error
    setMetaTitle(prevTitle)
    setMetaDescription(prevDesc)
    toast.error('Erreur reseau')
  } finally {
    setSaving(false)
  }
}
```

Do NOT change any other functionality in the file. Only modify `handleSave` and add `setGuide` to the store selector.
